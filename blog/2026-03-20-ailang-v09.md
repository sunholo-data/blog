## What is AILANG?

AILANG is a purely functional, effect-typed language designed as a deterministic execution substrate for AI-generated code. Every construct has deterministic semantics that can be reflected, verified, and serialized.

v0.9 is the biggest release since the language launched. It takes AILANG from a local development tool to a cloud-native platform -- you can now publish reusable packages, multiplex real-time data streams, and deploy HTTP APIs to Cloud Run with a single annotation.

## Share and Reuse Code with the Package Registry

You can now publish and install AILANG packages with two commands.

```bash
# Publish your package to the registry
ailang publish

# Install a package (always pins exact versions)
ailang install sunholo/auth@0.1.0

# Or grab the latest
ailang install sunholo/auth
```

The registry enforces exact version pinning -- no semver ranges, no `^` or `~` wildcards. When you run `ailang install sunholo/auth`, it resolves the latest version and writes the exact pin to your `ailang.toml`. Deterministic builds from day one.

Packages include SHA-256 content hashes and interface hashes in the lockfile. If a dependency changes on disk, the build catches it. Lockfiles are now portable too -- no more absolute paths, so they work in Docker and CI without re-resolving.

## Multiplex Real-Time Streams with Async I/O

AILANG v0.9 introduces a multi-source event multiplexer that lets you combine WebSocket connections, subprocess output, and stdin into a single deterministic event loop.

The multiplexer uses priority-ordered dispatch. Highest-priority sources win when multiple events arrive simultaneously, with round-robin within the same priority band to prevent starvation. When no events are ready, it blocks efficiently.

Subprocess streaming is built in. You can spawn a process, read its stdout as chunked events, and write to its stdin -- all within the same `selectEvents` loop. Processes are managed with proper lifecycle handling: SIGTERM, a 5-second grace period, then SIGKILL. No zombies.

```ailang
import std/stream (
  asyncExecProcess, sourceOfConn,
  selectEvents, StreamSource
)

// Multiplex a WebSocket and a subprocess in one loop
let micSource = asyncExecProcess("rec", ["-q", "-"], "mic", 1, 4096)
let wsSource = sourceOfConn(conn, "server", 2)

selectEvents([micSource, wsSource], \event.
  match event {
    SourceBytes(name, data) => handleAudio(data),
    SourceText(name, text)  => handleMessage(text),
    _                       => ()
  }
)
```

This is particularly useful for real-time applications -- ambient audio assistants, log aggregation, or any scenario where you need to combine multiple I/O sources without threads or callbacks.

## Deploy to Cloud Run with Route Annotations

You can now turn any AILANG module into an HTTP API with `@route` annotations. The serve-api runtime reads these at startup and registers the endpoints automatically.

```ailang
module myapp/api

import std/json (toJson)

@route("POST", "/api/v1/parse")
export func parseDocument(path: string) -> string ! {IO, FS} {
  // Your logic here
  toJson({status: "ok", path: path})
}

@route("GET", "/api/v1/health")
@nowrap
export func health() -> string ! {} {
  toJson({healthy: true})
}
```

Two new annotations give you fine-grained control over HTTP responses. `@nowrap` returns raw JSON without the standard `FunctionCallResponse` envelope -- essential for protocol-compliant endpoints like A2A agent cards or OpenID discovery documents. `@raw` gives your handler the full HTTP request context (headers, body, method, query) for webhooks that need signature verification.

Custom routes appear automatically in the generated OpenAPI spec and A2A Agent Card. The API supports multipart file uploads (up to 50MB by default), binary responses for file downloads, and API key authentication.

[Explore the serve-api docs](https://ailang.sunholo.com/docs/guides/serve-api)

## Debug Without the Noise

Debug is now a true ghost effect. You can call `Debug.log()` and `Debug.check()` anywhere without declaring `! {Debug}` in your function signature. The effect is invisible to callers.

In development, debug output collects and prints to stderr after execution. In production, `ailang run --release` erases all debug calls to `()` at compile time -- zero runtime cost.

This means packages don't need to list Debug in their effect ceilings, you don't need `--caps Debug` at runtime, and your function signatures stay clean. Just import the logger and it works.

## Grounded in 12 design axioms

Every feature in AILANG traces back to a set of [non-negotiable design axioms](https://ailang.sunholo.com/docs/references/axioms). Three that shaped v0.9 directly:

- **Axiom 7: Machines Are Primary Readers.** AILANG assumes AI models are the primary consumers of code. The package registry, lockfile format, and import system all prioritize decidable structure and machine analysis over syntactic convenience.
- **Axiom 4: Authority Must Be Explicit.** No program has implicit access to the world. The `@route` annotation system makes HTTP authority visible in the type signature — a `POST /api/v1/parse` endpoint declares `! {IO, FS}` right in its function type.
- **Axiom 9: Cost Is Part of Meaning.** Capability budgets (`AI @limit=20`) make resource consumption part of a program's semantics, not an incidental runtime detail.

## More in v0.9

**Relative imports** -- Use `import ./plan (Plan)` for sibling modules within the same package. Three-way import distinction: `./` for local, `pkg/` for external, `std/` for stdlib.

**`uuid4()` builtin** -- Generate RFC 4122 v4 UUIDs via `rand.uuid4()` with the `Rand` effect. Useful for API keys, request IDs, and correlation tokens.

**Named parameter binding** -- Agents can send `{"path": "file.docx", "output_format": "blocks"}` instead of positional args. Automatic snake_case to camelCase conversion.

**AILANG version gating** -- Packages declare their minimum AILANG version in `ailang.toml`. The installer checks compatibility before downloading.

**Iterative list builtins** -- `map`, `filter`, and `foldl` now run as Go builtins with O(1) stack usage. 50K elements in ~2ms. No more recursion limit at 10K+ items.

## Getting Started

Install AILANG and create your first package:

```bash
# Install AILANG
curl -fsSL https://ailang.sunholo.com/install.sh | bash

# Create a new package
ailang init package --dep sunholo/config

# Install dependencies
ailang install

# Run your module
ailang run --caps IO main.ail
```

If you're already using AILANG, upgrade to v0.9 and run `ailang lock` to regenerate your lockfile with the new portable format.
